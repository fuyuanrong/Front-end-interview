VUE和React比较

相同点

1.都是组件化开发都有Virtual DOM

2.都支持props进行父子组件间数据通信

3.都支持数据驱动视图,不直接操作DOM,更新状态数据界面就自动更新

4.都支持服务器端渲染

5.都有支持native的方案,React的 React Native ,Vue的Weex

不同点

1.数据绑定:vue实现了数据的双向绑定,react数据流动是单向的

2.组件写法不用,react推荐的是jsx(html,css都写进js,即'all in js);vue推荐的是单文件组件格式(webpack+vue-loader,即htmlcssjs写在一个文件内)

3.state对象在react中不可变,需要用setState方法更新,vue中,state对象不是必须的,数据由data属性在vue对象中管理

4.Virtual DOM不一样,vue会跟踪每一个组件的依赖关系,不需要重新渲染整个组件树,而React,每当应用状态改变时,全部组件都会重新渲染,React中需要shouldComponentUpdate这个生命周期函数方法来进行控制

5.React严格上只针对MVC的view层,VUE是MVVM模式
